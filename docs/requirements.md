# 需求文档

## 项目目标

建设一个 Android 综合媒体播放应用，使其与当前 UI 原型保持一致，并支持影视点播、剧集播放、IPTV 直播播放、电台、播客、在线 M3U8/M3U 链接播放和应用内更新检查。

## 用户角色

- 普通用户：浏览影视内容、搜索影片、播放剧集、观看电视直播、收听电台和播客、输入链接在线播放。
- 运营/维护人员：维护视频源、直播源、电台源、播客源、版本发布和更新入口。
- 开发/测试人员：验证 UI 还原、播放能力、源解析能力和发布更新链路。

## 功能需求

### 导航

- 应用底部必须包含 `片库`、`电视`、`音频`、`我的` 四个主入口。
- 剧集播放页和直播播放页不展示底部导航。
- 详情页当前保留底部导航，以匹配现有原型。

### 片库

- 展示来自已启用视频源的影视内容。
- 支持下拉刷新。
- 支持列表底部继续加载更多。
- 顶部提供搜索入口。
- 加载、空数据、失败状态必须有明确反馈。

### 搜索

- 支持输入关键词。
- 支持跳转到搜索结果页。
- 搜索结果可进入详情页。
- 搜索失败时展示可读错误信息。

### 详情页

- 海报和简介区域放在一排：左侧海报，右侧基础信息和简介。
- 播放源和播放线路控件应保持紧凑。
- 剧集列表直接在详情页展示，不再进入二级下钻页面。
- 剧集按钮保持紧凑，一行尽量展示更多集数。
- 不允许默认选中任意异常集数，例如默认选中第 12 集。
- 点击剧集后进入剧集播放页。

### 剧集播放

- 顶部展示返回按钮、影片名称和当前集数。
- 播放器下方展示当前播放源地址，并支持点击复制图标复制。
- 支持快退、播放/暂停、快进。
- 快退、播放/暂停、快进三个按钮宽度保持一致。
- 支持投屏按钮，并放在下方控制按钮组中。
- 支持全屏播放，且全屏画面不能显示不全。
- 支持倍速播放，最大 4 倍。
- 支持展示分辨率、码率、实时网速等播放参数。
- 离开播放页后必须销毁或释放播放器，避免后台声音和旧画面残留。

### 直播

- 支持从 M3U 源解析频道列表。
- 默认直播源为：

```text
https://raw.githubusercontent.com/lonnnnnng/iptv-api/refs/heads/master/output/user_result.m3u
```

- 搜索频道控件放在顶部。
- 直播源切换入口为小按钮，并放在分组筛选行左侧。
- 支持频道分组筛选。
- 点击频道进入直播播放页。

### 直播播放

- 顶部标题风格必须与剧集播放页一致：返回按钮加当前频道名。
- 播放器下方展示当前播放源地址，并支持复制。
- 投屏、播放/暂停、全屏放在一行，宽度相等。
- 展示分辨率、码率、实时网速等播放参数。
- 离开页面后必须释放播放器。

### 音频

- 聚合电台和播客两个入口。
- 电台支持源切换、搜索、分类筛选和播放。
- 播客支持 RSS/Atom 源聚合、搜索、节目列表和播放。
- 音频播放应支持通知栏和锁屏控制。

### 在线播放

- 在线播放入口位于 `我的` 页面。
- 页面保留必要说明文案。
- 页面只提供一个播放链接输入框和一个解析播放按钮。
- 输入有效 M3U8 链接后，进入剧集播放式播放页。
- 输入有效 M3U 链接后，解析频道并进入直播播放式流程。
- 输入无效链接时展示明确错误提示。

### 我的

- 提供播放历史入口。
- 提供在线播放入口。
- 提供视频源管理入口。
- 提供直播源管理入口。
- 提供电台源管理入口。
- 提供播客源管理入口。
- 提供重置应用能力。
- 提供免责声明。
- 提供检测更新按钮。
- 检测到新版本时，展示当前版本、新版本、更新说明、安装包大小和下载按钮。
- 下载更新时展示进度条。
- 下载完成后唤起 Android 系统安装器。
- 没有新版本时提示当前已是最新版本。
- 更新接口不可访问时展示清晰错误原因。

## 非功能需求

- UI 必须尽量 1:1 对齐当前原型。
- 页面元素不能重叠、挤压或异常形变。
- 网络请求不得阻塞主线程。
- 播放器生命周期必须可靠，离开页面后不能继续播放。
- 本地数据源和历史记录需要持久化。
- 用于覆盖安装的 APK 必须使用一致签名。

## 非目标范围

- DRM 播放。
- 用户账号体系。
- 云端同步。
- 会员、支付或内容权益系统。

## 已知限制

- 应用内更新依赖公开 GitHub Release、APK asset 和一致签名。
- 第三方视频源和直播源可能不可用或格式变化。
- 直播稳定性取决于上游 M3U 源质量。
