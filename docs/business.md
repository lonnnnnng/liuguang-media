# 业务文档

## 产品定位

流光是一个 Android 综合媒体播放客户端，用于整合配置好的影视点播源、IPTV 直播源、电台源、播客源和用户输入的播放链接。它的目标不是生产内容，而是提供统一、紧凑、易操作的移动端播放体验。

## 核心价值

- 将影视点播、电视直播、音频聚合和在线链接播放整合在同一个应用里。
- 降低用户在多个播放器或工具之间切换的成本。
- 为运营人员提供视频源、直播源、电台源和播客源维护入口。
- 通过 GitHub Actions 自动构建 APK，降低手工打包成本。

## 核心业务流程

### 观看剧集

1. 用户进入 `片库`。
2. 用户浏览最近更新或通过搜索查找内容。
3. 用户进入详情页。
4. 用户查看海报、基础信息、简介、播放源、播放线路和剧集列表。
5. 用户点击某一集。
6. 应用进入剧集播放页并开始播放。
7. 用户可进行快退、播放/暂停、快进、倍速、全屏、复制源地址等操作。
8. 用户返回后，播放器释放资源。

### 观看直播

1. 用户进入 `电视`。
2. 应用从已启用 M3U 源解析频道。
3. 用户搜索频道、切换直播源或筛选分组。
4. 用户点击频道。
5. 应用进入直播播放页。
6. 用户可播放/暂停、投屏、全屏、复制源地址并查看播放参数。

### 在线链接播放

1. 用户进入 `我的`，点击 `在线播放`。
2. 用户输入 M3U8 或 M3U 链接。
3. 用户点击解析播放。
4. M3U8 直接进入播放页。
5. M3U 解析为频道后进入直播播放流程。

### 收听音频

1. 用户进入 `音频`。
2. 用户在电台和播客之间切换。
3. 用户搜索或筛选内容。
4. 用户点击电台或播客节目开始播放。
5. 应用通过通知栏和锁屏媒体控制继续提供播放控制。

### 应用更新

1. 用户进入 `我的`。
2. 用户点击 `检测更新`。
3. 应用请求最新 GitHub Release。
4. 如果有新版本，弹窗展示版本信息、更新说明和下载入口。
5. 用户下载 APK。
6. 下载完成后唤起系统安装器。

## 数据源运营

当前默认视频源：

- 无尽资源
- 量子资源
- 非凡资源

当前默认直播源：

- IPTV直播源
- 播放测试源

当前默认音频源包含网络电台源和播客源，源可用性由上游服务决定。

数据源的稳定性直接影响首页内容、搜索结果、剧集播放和直播播放体验。运营人员需要定期检查数据源是否可访问、返回格式是否变化、播放链接是否可用。

## 成功标准

- 应用可在目标模拟器启动。
- 片库刷新和加载更多可用。
- 搜索、详情、剧集播放主链路可用。
- 电视直播源解析、频道筛选、直播播放可用。
- 音频页电台和播客播放可用。
- 在线 M3U8/M3U 解析播放可用。
- 离开播放页后无后台声音和旧画面残留。
- GitHub Actions 能成功产出 APK。
- GitHub Release APK 在签名一致时可覆盖安装旧版本。

## 风险

- 上游视频源不可用会导致内容加载失败。
- 上游直播源不可用会导致频道为空或播放失败。
- Release 未发布、APK 缺失或签名不一致会影响应用内更新。
- APK 签名不一致会导致 Android 拒绝覆盖安装。
- 网络波动会影响播放、M3U 解析和网速显示准确性。

## 合规说明

应用只提供播放客户端和源配置能力。影视、直播内容的版权、合法性和可分发性需要由项目运营方在发布前自行确认。
